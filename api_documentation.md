# Book API purpose
The Book API is needed to allow users to manage their personal digital library.

The client application requires a structured way to:

-- store books
-- retrieve books
-- update book information
-- delete books

Without an API, the client would not be able to communicate with the server in a consistent and scalable way.

# Book API solves

The API solves the problem of:

-- organizing user data (books) in a structured way
-- enabling CRUD operations (Create, Read, Update, Delete)
-- separating client and server responsibilities

This allows the frontend to remain simple while the backend handles data logic.

# User API purpose

The User API is needed to manage user accounts within the application.

It allows users to:

-- create an account
-- delete their account

This is essential for providing personalized access to the digital library.

# User API solves

The API solves the problem of:

-- identifying users in the system
-- securing access to personal data
-- allowing users to control their own data

It ensures that each user can manage their own account independently.
